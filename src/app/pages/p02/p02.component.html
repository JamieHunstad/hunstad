A daily Earth imaging application utilizing the NASA “EPIC” API. The EPIC API provides information on the daily imagery collected by DSCOVR's Earth Polychromatic Imaging Camera (EPIC) instrument. This was a personal project I created in my spare time to challenge my skills in Angular!

Brief

This project came from my desire to work with an API from NASA, my only reason - I thought it'd be FUN! My goal was to create an application within Angular to serve a daily image of the Earth with past search functionality.

Ideation

Sketching allowed me to explore many possibilities within layout and functionality. It was important that both desktop and mobile concepts were explored so that development would be quicker later on. Through ideation, I found that I wanted to focus on the "change date" feature. I chose this because I felt it would be a good challenge to work with API endpoints and template-driven forms.

Development

At this stage in the process, I began to create my project using Angular. I chose Angular because it would allow me to test my skills in the framework. I used the NASA “EPIC” API to display the image. There were two endpoints I was interested in - the date and the image endpoints. The date endpoint would provide JSON data with the image name, while the image endpoint would use that image name to display the image. The user could then specify the date on which they would like to see it. This was the basis for how the app would function.

I create an API call using the HTTP client service. I use the HTTP .get() method to fetch data from the NASA "EPIC" API. I use the "myDate" parameter (specified in the service) to call the specific date.

Development did not go without its issues. I found it would be crucial that the app had functionality to notify when an image was not available. The spacecraft does not always take images due to the camera being down. This meant I had to specify in the code that if an image was not found, the alert would be presented.

Having a “Loading” feature also became important. It takes time for the image to load from the API, meaning that users would need feedback to know whether the application was working. To combat this, I added code within the initial API call to display the “Loading” text and removing once the API call is complete.

Conclusion

This project was a fun challenge to test my skills in Angular and API knowledge. The API came with its obstacles but pushed me to find solutions, helping me become a better developer!
